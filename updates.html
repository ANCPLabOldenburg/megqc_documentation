Update notes

What changed in each release.

A plain-language history of every MEEGqc release on PyPI, newest first. Each card lists the new features, the bug fixes, and anything worth knowing before you upgrade. Use the menu on the left to jump straight to a version.

▸
Package names and how to read these notes

The tool is published on PyPI as meg-qc and imported in Python as meg_qc. Since version 1.0.0 a thin meeg-qc package installs exactly the same thing under the MEEGqc name. Upgrade any time with pip install --upgrade meg-qc (or meeg-qc). Version numbers match the PyPI release history. Labels are colour-coded: New Fixed Notes

1.0.9

8 July 2026 Latest

New

  • A "Version notes" button in the app. The main window now has a Version notes button next to Check updates, in the bottom row. Clicking it opens this page in your browser, so you can always see what changed in each release without leaving MEEGqc.

1.0.8

8 July 2026

A large clean-up release that makes every file MEEGqc writes follow the BIDS naming rules, unifies the units on all figures, and renames a few things for clarity.

New

  • BIDS-valid file names everywhere. Result files now use plain letters-and-numbers labels (for example desc-STDs, desc-PSDs, desc-NoisyEpochsOnSTDBaseMag) so any BIDS tool can read them. Reports and Global Quality Index (GQI) files were renamed the same way. The names are now written correctly from the start, with no behind-the-scenes renaming step.
  • The derivatives folder is now derivatives/MEEGqc/ (it used to be Meg_QC), because the tool handles MEG and EEG.
  • Clearer report names. "group" reports are now "dataset" reports and "multisample" is now "multi-dataset". This shows up in the command-line flags (--qa-dataset, --qc-dataset, --qa-multi-dataset, --qc-multi-dataset), the GUI checkboxes, the Python function names, and the report file names.
  • The same physical units on every figure. Magnetometers are shown in femtotesla (fT), planar gradiometers in fT/cm, axial gradiometers in fT, and EEG, ECG and EOG in microvolts (uV). Power spectra use amplitude spectral density (for example fT per square-root Hz). This also fixed EEG being scaled wrongly while still labelled in microvolts.
  • Colour-map controls on the dataset and multi-dataset reports too. Their 3D sensor maps now get the same colour-map switcher (Red-Blue by default, plus Turbo, Jet, Viridis, Plasma, Reds) that the subject report already had.
  • A Settings tab in the dataset and multi-dataset reports, showing the exact settings that produced each report, with a tidy "machine-readable input" box.
  • Multi-dataset report titles now list only the datasets that are actually in that report.

Fixed

  • Dataset and multi-dataset report file names are now readable by BIDS tools (no dataset names or timestamps glued together with underscores).
◆
Upgrading from an older version

This is a breaking change. Reports made by this version cannot read result files produced by older versions. Re-run the calculation step to regenerate them. MEEGqc detects an old Meg_QC folder and tells you to re-run rather than failing silently.

1.0.7

24 June 2026

New

  • Interactive 2D sensor maps. A new way of drawing sensor maps, flattened onto a 2D head outline (with nose and ears), shown next to the existing 3D maps as side-by-side (2D) and (3D) tabs. It works for any MEG system and any EEG montage, and covers the STD, PtP, PSD, ECG and EOG metrics, plus a plain 2D sensor-layout view.
  • Per-figure controls: a View control to change the map shape (flattened round, field shape, or fitted to head), and a colour-map switcher (Red-Blue by default with high in red and low in blue, plus Turbo, Jet, Viridis, Plasma, Reds).
  • A Settings tab in the subject report listing the exact settings used, and the software version stamped on the report.

Notes

  • The reports now show the MEEGqc name instead of MEGqc.

1.0.6

23 June 2026

Fixed

  • The settings editor now accepts decimal numbers for epoch_tmax (for example 0.8), not only whole numbers.

1.0.0 – 1.0.5

27 May 2026

New

  • Rebrand to MEEGqc, to reflect that the tool now handles both MEG and EEG. A new meeg-qc package on PyPI installs the same code (it pins the exact matching meg-qc version), so pip install meeg-qc and pip install meg-qc give you the same thing. Every original command keeps working, and matching meeg* names were added alongside them.
  • The graphical interface, window titles, About dialog and log messages were rebranded to MEEGqc, with a one-time settings migration so your preferences carry over on first launch.
  • The in-app updater was modernised: it only offers stable releases, is safe on Windows (where files can be locked while in use), and knows whether you installed meg-qc or meeg-qc.
  • Linux application icons shipped, the README was rewritten for MEEGqc, and a citation file (CITATION.cff) was added.

Notes

  • Versions 1.0.1 to 1.0.5 were same-day packaging and metadata fixes for the 1.0 release.

0.9.8

15 April 2026

New

  • Runs on Python 3.10 through 3.14. The install now picks the newest version of each dependency that works with your Python, instead of pinning one exact set.
  • numba (used to speed up one metric) is now optional: if it is not installed, MEEGqc falls back to a slower but working path instead of crashing.

Fixed

  • The built-in report viewer handles very large interactive reports (it serves reports over 5 MB through a small local web server) and can read local files correctly.
  • Compatibility fixes for newer pandas and Plotly, plus a fix for maps where every value is identical.
  • On Linux, the app now auto-resolves a common missing system library (libxcb-cursor0) and works around graphics-driver failures in the built-in report viewer.

0.9.5 – 0.9.7

10 April 2026

New

  • EEG channels are now processed by default.
  • The built-in report viewer uses a web engine, with an automatic fallback to your normal browser if that fails.

Fixed

  • Removed the old automatic Peak-to-Peak metric; only the manual Peak-to-Peak (PtP) remains.
  • Windows text-encoding (UTF-8) fix.
  • Windows: worked around a crash caused by empty or unreadable JSON sidecar files.

0.9.0 – 0.9.4

2 – 8 April 2026

New

  • Full EEG support. MEEGqc now reads the common EEG formats (EDF, BDF, BrainVision, EEGLAB, EGI/MFF, Neuroscan) and every quality metric works on EEG channels. It detects the modality from the BIDS file, auto-detects the EEG montage, standardises the reference, corrects channel types from channels.tsv, and uses EEG-specific thresholds and frequency bands.
  • A new QC Viewer: an interactive signal viewer built into the app for looking at the raw recording and the quality results.
  • New GUI tools: open a command-line window and a live terminal from inside the app; select several dataset folders at once.
  • Improved one-click installers for macOS, Linux and Windows, and new icons.

Notes

  • Because MEG and EEG are now both supported, derivatives are organised into meg/ and eeg/ subfolders and the settings-snapshot file was renamed. Two new dependencies were added for the viewer (pyqtgraph and PyQt6-WebEngine).

0.7.9

25 March 2026

New

  • Configurable filtering and resampling. You can now turn filtering off (apply_filtering = False) and turn resampling off (downsample_to_hz = False).
  • Choose your own trigger channels. The stimulus channels are no longer hardcoded; set preferred_stim_channels and noisy_stim_channels in the settings, keeping spaces in names like STI 101.
  • Transparent event detection. The report now shows where events came from (the BIDS events file, trigger detection, or fixed-length windows) and how many events were found and used, with a per-event-type table.
  • A new Event summary file per recording that records the detected events and compares them with the BIDS events file.
  • A toggle to keep heatmap channels in their natural order.

Notes

  • Version 0.7.9b1 (24 March 2026) was the test pre-release of the same changes.

0.7.5 – 0.7.8

12 – 20 March 2026

Fixed

  • A series of quick follow-up releases that hardened output-path handling across Windows, macOS and Linux so reports and derivatives always land in the right place.

Notes

  • Added documentation for running MEEGqc on high-performance computing (HPC) clusters through CBRAIN.

0.7.0 – 0.7.4

6 – 12 March 2026

New

  • One consistent way to run MEEGqc from the command line, the GUI and Python. All three now go through the same code, so they behave identically.
  • Analysis profiles for reproducible re-runs. A mode control (legacy, new, reuse, latest) with a profile name lets you re-run the same dataset with different settings and keep the results side by side, instead of overwriting them. Good for checking how sensitive your results are to a threshold.
  • Run everything in one command. A single run-megqc-plotting command can render every report scope (subject, dataset and multi-dataset, both QA and QC), and a "run all" path does calculation and every report in one go. Per-dataset overrides let you use different settings, subjects or worker counts for each dataset.
  • Redesigned subject reports, a new dataset-level QC report, on-page controls (font size, dot size, line thickness) on every report, a summary-distribution tab, faster loading of large reports, and a GUI auto-update feature.

Fixed

  • Reports are now aware of separate runs, with a safe fallback when a BIDS dataset has no dataset_description.json.
  • Per-subject, per-metric error reporting, so one problematic recording or one failing metric no longer stops the whole run.
  • Output-path handling fixed for locked-down "corporate" computers.

0.6.1 – 0.6.8

August 2025 – March 2026

The follow-up series that steadily polished the reports and broadened platform support after the 0.6.0 rework.

  • 0.6.1: revamped summary QC report with interactive tables, and a lighter, configurable report back-end; ECG channel plotting fix.
  • 0.6.2: plotting-module fix.
  • 0.6.3: more reliable report generation (better handling when result files lack task or run labels) and sturdier background workers in the GUI.
  • 0.6.4: maintenance release.
  • 0.6.5: macOS support, an automated test for the MEG pipeline, and PyQt6 dependency fixes.
  • 0.6.6: you can now send derivatives to a custom output folder instead of inside the dataset.
  • 0.6.7: handles split FIF files, logs recording duration, resolves external derivative paths for plotting and GQI, supports CTF gradiometer-only GQI summaries, and adds a task column to the GQI table.
  • 0.6.8: final checkpoint of the 0.6 series, just before the 0.7.0 overhaul.

0.6.0

4 August 2025 Landmark release
▸
The release that made MEEGqc usable on a normal computer

Up to and including 0.5.12, MEEGqc loaded whole recordings into memory and processed them one after another. On an ordinary laptop that meant it often ran out of memory or was simply too slow to use. Version 0.6.0 reworked the engine to write intermediate results to disk instead of holding everything in memory, and to process subjects in parallel across the computer's CPU cores. The result was a large drop in memory use and a large jump in speed, so the tool finally ran comfortably on everyday machines. This is the single most important release in the project's history.

New

  • Parallel processing and on-disk memory management. Subjects are processed in parallel and heavy intermediate data is written to disk, so MEEGqc no longer needs to hold an entire recording in memory. This is what made it fast and usable on normal computers.
  • The first graphical interface (GUI). A desktop app to choose datasets, edit settings, run the analysis and open the reports, so you no longer need the command line.
  • The first Global Quality Index (GQI). A single, configurable quality score per recording built from the individual metrics, with adjustable penalty weights, an option to turn it off, a dataset-level summary table, and the ability to recompute it on its own without redoing the whole analysis.
  • Dataset-level reports with lazy loading so large group reports open quickly, plus the first cross-dataset comparison tools.
  • Support for CTF recordings in the new optimised pipeline, temporary-file support for Windows machines, and saved configuration files that work with the parallel-processing approach.

Notes

  • This release gathered roughly eight months of work (there were no PyPI releases between 0.5.12 in December 2024 and 0.6.0 in August 2025). Everything above first reached users here.

Early releases

0.0.3 – 0.5.12, May 2023 – December 2024

The foundational years. MEGqc began as a research toolbox and was first published to PyPI in May 2023 (0.0.3). Across the 0.0.x to 0.5.x series it grew the core quality metrics (standard deviation, peak-to-peak amplitude, power spectral density, ECG and EOG correlation, muscle noise and head movement), the first automatic HTML reports, and a command-line workflow. At this stage the tool ran on a single CPU core and kept whole recordings in memory, which is exactly what the 0.6.0 rework above fixed. There was no graphical interface and no Global Quality Index yet. These releases were rapid and experimental; systematic, per-version notes begin at 0.6.0.
